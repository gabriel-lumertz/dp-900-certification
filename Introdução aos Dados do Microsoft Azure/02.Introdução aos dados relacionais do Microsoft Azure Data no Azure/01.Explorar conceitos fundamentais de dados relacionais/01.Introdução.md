# Introdução

Nos primeiros anos de existência dos sistemas de computação, todos os aplicativos armazenavam dados nas próprias estruturas exclusivas deles. Quando os desenvolvedores queriam criar aplicativos para usar esses dados, eles tinham que saber muito sobre a estrutura de dados específica para encontrar os dados de que precisavam. Essas estruturas de dados eram ineficientes, difíceis de manter e difíceis de otimizar para um bom desempenho do aplicativo. O modelo de banco de dados *relacional* foi projetado para resolver o problema de várias estruturas de dados arbitrárias. O modelo relacional fornece um padrão de representar e consultar dados que podem ser usados por qualquer aplicativo. Uma das principais vantagens do modelo de banco de dados relacional é o uso de *tabelas*, que são uma maneira intuitiva, eficiente e flexível de armazenar e acessar informações estruturadas.

O modelo relacional simples mas poderoso é usado por organizações de todos os tipos e tamanhos para uma ampla variedade de necessidades de gerenciamento de informações. Os bancos de dados relacionais são usados para controlar inventários, processar transações de eCommerce, gerenciar quantidades enormes de informações críticas do cliente e muito mais. Um banco de dados relacional é útil para armazenar informações que contêm elementos de dados relacionados que precisam ser organizados em uma estrutura consistente e baseada em regras.

Neste módulo, você aprenderá sobre as principais características dos bancos de dados relacionais e explorará as estruturas de dados relacionais.
